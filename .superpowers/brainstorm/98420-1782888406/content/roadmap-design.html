建议开发节奏：5 周完成内部可用 MVP

按 2 名前端 + 1 名后端估算；若 1 人全栈，建议按 8–10 周计算。

第 1 周
骨架与最小闭环
Monorepo、Project Schema v1、三栏编辑器、栅格拖拽、柱状图 + 文本、撤销/重做基础。
验收：拖入组件、移动缩放、刷新后恢复。
第 2 周
数据闭环
CSV/Excel 上传解析、统一接口查询、字段类型推断、数据预览、字段到图表槽位的映射与校验。
验收：一份文件和一个 API 数据集均可驱动柱状图。
第 3 周
组件与配置体系
8 个组件全部接入注册表；Schema 驱动字段/样式面板；统一主题、空态、加载态和错误态。
验收：无需改编辑器主体即可注册新组件。
第 4 周
持久化与预览
项目保存/加载、自动保存、独立预览页、Schema 版本迁移、接口权限透传和查询超时处理。
验收：编辑态与预览态渲染结果一致。
第 5 周
稳定性与试用
核心 E2E、性能基线、1 万行数据验证、异常恢复、埋点、内部用户试用与缺陷收口。
验收：核心路径通过，P0/P1 缺陷清零。

技术栈

  • pnpm + Turborepo
  • React / Vite / TypeScript
  • Ant Design + ECharts
  • react-grid-layout + Zustand
  • TanStack Query + Zod

服务端与质量

  • NestJS + Fastify
  • Prisma + PostgreSQL
  • Vitest + Testing Library
  • Playwright E2E
  • OpenAPI + 统一错误码
