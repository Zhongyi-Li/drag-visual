# BI 看板前端后续开发建议

## 一、当前已完成能力

当前前端已经具备 BI 看板编辑器的核心雏形：

- 三列式编辑器布局：左侧图表组件库、中间画布、右侧配置栏。
- 图表组件拖拽添加、选中、复制、删除、撤销、重做。
- 右侧配置栏支持折叠，折叠后画布区域可向右延展。
- 支持上传 CSV / Excel 文件并解析为本地数据集。
- 图表配置面板可选择数据集和绑定字段。
- 空数据状态下，不同图表类型已有对应 demo 占位展示。
- 明细表已支持真实 table 表头、中文字段名和分页。
- 基础图表渲染链路已打通，包括柱图、折线图、饼图、KPI、明细表等。

## 二、下一阶段建议优先级

建议下一阶段优先推进三条主线：

1. 数据集管理完善
2. 图表配置项开发
3. 保存、预览、发布闭环完善

这三条完成后，产品会从“可搭建 demo”进入“可交付给业务试用”的阶段。

## 三、数据集管理完善

当前已经能上传并解析文件，但还缺少完整的数据集管理能力。

建议继续开发：

- 上传成功后展示数据预览。
- 支持修改数据集名称。
- 支持删除、替换已上传数据集。
- Excel 支持多 Sheet 选择。
- 字段类型识别后允许手动调整。
- 支持字段显示名编辑。
- 导入异常提示更明确，例如空表、重复表头、格式错误、文件过大。
- 上传数据持久化，避免刷新页面后丢失。
- 后续接后端上传接口，把文件数据保存到服务端。

## 四、图表真实渲染能力补齐

当前部分图表已经有真实渲染，其余图表还主要停留在空状态 demo 或基础占位。

建议按图表类型逐步补齐：

- 面积图、堆积图、百分比图。
- 条形图、排行榜、瀑布图、漏斗图。
- 仪表盘、水波图、进度条。
- 雷达图、矩形树图、热力图。
- 表格类组件继续完善排序、固定表头、分页大小配置、列宽和空值展示。

## 五、图表配置项开发

右侧配置栏当前主要承担数据绑定，后续需要扩展为完整图表配置面板。

建议继续开发：

- 标题配置。
- 颜色配置。
- 图例显示 / 隐藏。
- 坐标轴配置。
- 数据标签显示 / 隐藏。
- 数值格式化，例如金额、百分比、小数位。
- KPI 聚合方式，例如求和、平均、最大值、最小值。
- 表格分页大小、是否显示序号、是否开启斑马纹。

## 六、数据处理能力

业务数据接入后，图表不能只绑定字段，还需要支持常见数据处理。

建议继续开发：

- 排序。
- 过滤。
- Top N。
- 分组聚合。
- 多指标展示。
- 日期字段按日、周、月聚合。
- 空值处理。
- 字段别名。
- 计算字段。

## 七、编辑器交互打磨

当前编辑器已经具备基础拖拽编辑能力，后续重点是提升编排效率和稳定性。

建议继续开发：

- 拖拽吸附线。
- 防重叠碰撞规则细化。
- 组件尺寸最小值限制，避免图表压缩后展示异常。
- 多选组件。
- 批量移动。
- 组件对齐。
- 画布缩放。
- 快捷键提示。
- 组件复制、删除、撤销体验优化。

## 八、保存、预览、发布闭环

这是 BI 看板从编辑器走向产品交付的关键闭环。

建议继续开发：

- 保存看板配置到后端。
- 保存上传的数据集引用。
- 保存图表字段绑定和图表属性配置。
- 预览页按当前草稿实时渲染。
- 发布生成只读看板页面。
- 发布后可重新进入编辑。
- 保存失败、版本冲突、发布失败的交互完善。

## 九、稳定性和性能

随着文件数据和图表数量增加，前端需要提前补充稳定性能力。

建议继续开发：

- 大 Excel 文件解析性能优化。
- 文件大小和行数限制。
- 表格虚拟滚动。
- 图表渲染防卡顿。
- 上传解析 loading 和取消。
- 错误边界。
- 关键链路单元测试和交互测试补齐。

## 十、建议排期顺序

建议后续按以下顺序推进：

1. 完善数据集管理，包括预览、重命名、删除、替换、字段类型调整。
2. 补齐常用图表真实渲染，包括面积图、条形图、仪表盘、热力图等。
3. 开发右侧图表配置项，包括标题、颜色、图例、坐标轴、数值格式。
4. 开发数据处理能力，包括排序、过滤、Top N、聚合。
5. 打通保存、预览、发布、再次编辑的完整闭环。
6. 做交互细节和性能优化，提升业务人员长期使用体验。
