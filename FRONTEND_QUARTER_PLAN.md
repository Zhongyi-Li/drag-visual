# Drag Visual 前端开发季度工作计划

## 一、计划定位

本计划用于下个季度前端开发工作报告与排期说明，周期按 3 个月、12 周规划。计划重点不是一次性完成完整产品，而是按照可交付、可验收、可演示的方式渐进推进前端能力建设。

规划范围仅包含前端工作：

- 前端页面框架与编辑器布局。
- 可视化画布、拖拽、吸附、防重叠碰撞。
- 图表组件封装、空状态、配置项开发。
- 数据绑定、接口接入、真实数据调测。
- 与后端联动保存配置、预览、发布。
- 前端交互细节、体验 polish、测试与验收。

不纳入本计划的内容：

- 后端服务实现本身。
- 登录、权限、组织、用户体系。
- 数据库建模、数据仓库、复杂数据分析能力。
- 商业化、计费、生产运维。

## 二、季度交付结果

本季度计划交付一套面向运营、结算、业务分析人员使用的 BI 看板前端能力，支持业务数据接入、图表展示、页面编排、配置保存和发布查看，满足内部经营分析、日常监控和汇报展示的基础需求。

季度末主要交付结果如下：

- 形成 BI 看板搭建基础能力：支持组件选择、拖拽编排、缩放调整和基础布局管理。
- 形成业务数据展示能力：支持数据集接入、字段绑定，并以图表、指标卡、表格等形式展示运营和结算数据。
- 形成看板配置管理能力：支持图表配置、字段配置、保存回显和发布前预览。
- 形成发布查看闭环：支持发布只读看板页面，并支持发布后再次进入编辑调整。
- 形成基础稳定性保障：覆盖空数据、接口异常、字段变化、保存失败等关键异常提示，为后续图表扩展和高级分析能力打基础。

## 三、季度总体目标

本季度目标是将当前 Drag Visual 前端从基础 MVP 推进到可用于阶段演示和内部验收的前端版本。排期上将原前三个基础阶段合并为第 1 阶段，并在第 1-2 周内完成，尽早形成可拖拽、可配置、可展示的编辑器底座，把更多周期留给数据集管理、真实数据绑定、保存发布闭环和验收打磨。用户应能够完成以下主流程：

1. 进入看板编辑器。
2. 在三列布局中选择图表组件。
3. 将组件拖拽到画布，并进行位移、缩放、吸附和防重叠布局。
4. 查看不同图表组件的空状态 demo。
5. 配置图表组件的数据字段和基础展示属性。
6. 接入接口数据并展示真实查询结果。
7. 与后端联动保存看板配置。
8. 预览和发布看板。
9. 在后续阶段完成细节交互、异常状态、视觉一致性和性能打磨。

## 五、阶段 1：编辑器基础、画布交互与图表组件体系

时间：第 1-2 周

### 目标

完成前端编辑器基础页面结构、核心画布交互和图表组件体系，让后续配置、数据绑定、接口联调和发布能力有稳定承载空间。本阶段由原前三个基础阶段合并而来，按快速底座建设执行，优先保证可演示、可测试、可继续扩展；复杂动效、精细视觉 polish 和复杂图表增强后移到验收打磨阶段。

### 功能模块拆分

#### 1. 应用路由与页面骨架

- 建立首页、编辑器、预览页、发布查看页等基础路由。
- 编辑器页面采用三列结构：
  - 左侧：图表组件面板。
  - 中间：可视化编辑画布。
  - 右侧：组件配置面板。
- 顶部增加工具栏：
  - 返回。
  - 看板名称。
  - 保存状态。
  - 撤销/重做。
  - 预览。
  - 保存。
  - 发布。

#### 2. 三列布局开发

- 左侧组件栏固定宽度，支持独立滚动。
- 中间画布区域自适应剩余空间。
- 右侧配置面板固定宽度，支持独立滚动。
- 处理浏览器窗口缩放时的布局稳定性。
- 保证编辑器最小可用宽度，避免内容挤压变形。

#### 3. 基础状态管理

- 建立编辑器 store。
- 管理当前看板数据。
- 管理选中组件。
- 管理 dirty 状态、保存中状态、保存失败状态。
- 预留 undo/redo 历史栈。

#### 4. 基础 UI 规范

- 统一编辑器背景、卡片边框、面板间距。
- 统一按钮尺寸、图标按钮、禁用态。
- 定义选中态、hover 态、空画布提示样式。

#### 5. 网格画布开发

- 中间画布采用 12 栅格布局。
- 设定统一 row height、margin、padding。
- 每个组件具备 x、y、w、h 布局信息。
- 布局数据与看板 schema 保持一致。

#### 6. 拖拽添加组件

- 支持从左侧组件面板拖拽组件到画布。
- 支持点击组件快速添加到画布。
- 根据鼠标释放位置计算组件落点。
- 超出边界时自动 clamp 到可用区域。
- 新增组件后自动选中。

#### 7. 组件移动与吸附

- 支持组件在画布内拖动。
- 拖动过程中按网格吸附。
- 拖动结束后写入最终布局。
- 拖动过程中显示占位反馈。
- 防止组件拖出画布边界。

#### 8. 防重叠碰撞

- 组件移动时避免与其他组件重叠。
- 组件缩放时避免覆盖相邻组件。
- 发生冲突时自动寻找可用位置或阻止非法尺寸。
- 保证布局数据不会出现重复占位。

#### 9. 缩放与尺寸约束

- 支持组件右下角缩放。
- 每类组件定义默认尺寸和最小尺寸。
- 缩放时遵守网格单位。
- 小尺寸下隐藏或收敛部分非关键内容，避免视觉溢出。

#### 10. 组件操作

- 选中组件。
- 复制组件。
- 删除组件。
- 撤销/重做。
- 快捷键删除。
- 快捷键保存。
- 操作按钮阻止拖拽冒泡。

#### 11. 组件注册体系

- 建立组件 registry。
- 每个组件定义：
  - 组件类型。
  - 显示名称。
  - 所属分类。
  - 默认布局。
  - 默认配置。
  - 数据槽位。
  - 配置 schema。
- 左侧组件面板根据 registry 渲染，而不是硬编码散落在页面里。

#### 12. 图表渲染器封装

- 封装统一 `DashboardComponentRenderer`。
- 图表类型通过统一入口渲染。
- 编辑态、预览态、发布态尽量复用同一渲染器。
- 渲染器内部区分：
  - 空数据状态。
  - 有数据状态。
  - 异常状态。
  - 文本组件状态。

#### 13. 图表空状态 demo

- 不同图表类型提供不同占位 demo。
- 空状态统一显示“当前图表无数据”。
- 空状态 demo 保持浅灰、弱化、不可交互。
- 需要覆盖的空状态类型：
  - 折线图。
  - 面积图。
  - 柱图。
  - 条形图。
  - 堆积图。
  - 百分比图。
  - 漏斗图。
  - 瀑布图。
  - 子弹图。
  - 箱形图。
  - 饼图。
  - 玫瑰图。
  - 雷达图。
  - 水波图。
  - 仪表盘。
  - 指标卡。
  - 进度条。
  - 表格。
  - 热力图。

#### 14. 基础图表数据态

- 使用 ECharts 或自定义 SVG/HTML 渲染基础图表。
- 支持柱图、折线图、饼图、KPI、表格的基础数据展示。
- 预留更多图表展示形态扩展点。
- 对不支持的复杂图表先提供合理占位和配置入口。

#### 15. 图表尺寸适配

- 大尺寸卡片显示完整 demo。
- 小尺寸卡片压缩图形和文案。
- 避免图形溢出组件卡片。
- 保证提示文案不遮挡标题栏和操作按钮。

### 主要产出

- 图表组件 registry。
- 统一图表渲染器。
- 各类图表空状态 demo。
- 基础图表数据态渲染能力。
- 前端状态管理基础完成。
- 左侧组件面板到画布的完整添加链路。
- 组件选中、复制、删除、撤销、重做能力。

### 验收标准

- 打开编辑器后能看到完整三列布局。
- 窗口缩放后布局不明显错位。
- 用户能连续添加多个组件且不会重叠。
- 拖拽和缩放结束后布局正确保存到前端状态。
- 复制/删除/撤销/重做结果符合预期。
- 组件操作按钮不会触发误拖拽。
- 从左侧添加不同类型图表后，空状态展示不再全部相同。
- 水波图、仪表盘、漏斗图、柱图、线图、饼图、表格等有可区分的占位效果。
- 编辑态和预览态展示风格一致。
- 没有数据时不会出现白屏或空白卡片。

## 六、阶段 2：组件配置项、数据集管理与数据绑定开发

时间：第 3-5 周

### 目标

让用户能够在右侧配置面板中配置图表数据来源、字段绑定和基础展示属性，使图表从静态占位逐步变成可配置组件。基础底座合并到阶段 1 后，本阶段承担更多产品化能力，包括本地数据集管理、数据预览、字段元数据调整和基础配置项完善。

### 功能模块拆分

#### 1. 配置面板结构

- 右侧配置面板拆分为：
  - 组件配置。
  - 数据绑定。
  - 样式配置。
  - 主题配置。
- 未选中组件时展示空状态。
- 选中文本组件时展示文本配置。
- 选中图表组件时展示数据和属性配置。

#### 2. 数据集选择

- 支持本地文件数据集导入与管理。
- 上传成功后展示数据预览。
- 支持修改数据集名称。
- 支持删除、替换已上传数据集。
- 字段类型识别后允许手动调整。
- 支持字段显示名编辑。
- 上传数据本地持久化，避免刷新页面后丢失。
- 接入数据集列表接口。
- 支持选择数据集。
- 选择数据集后加载字段 schema。
- 数据集加载中、加载失败、无数据集时给出反馈。

#### 3. 字段绑定

- 根据组件数据槽位展示字段选择项。
- 常见槽位包括：
  - 维度字段。
  - 指标字段。
  - 多指标字段。
  - 表格字段。
  - 分类字段。
  - 数值字段。
- 根据字段类型过滤可选项。
- 必填槽位未配置时给出提示。

#### 4. 组件可配置项

- 开发通用配置项：
  - 标题。
  - 显示图例。
  - 颜色。
  - 数值精度。
  - 前缀/后缀。
  - 聚合方式。
- 开发图表差异化配置：
  - 折线图：是否面积、是否平滑。
  - 柱图：方向、堆积、百分比。
  - 饼图：环形、玫瑰展示。
  - KPI：聚合、格式化、单位。
  - 表格：字段列、分页条数。
- 配置变更实时反映到画布。

#### 5. 配置校验与提示

- 字段类型不匹配时提示。
- 必填字段缺失时提示。
- 数据集 schema 变化时提示用户检查绑定。
- 清除绑定后恢复空状态 demo。

### 主要产出

- 可用的数据绑定面板。
- 本地数据集管理闭环。
- 基础组件属性配置面板。
- 字段类型校验与错误提示。
- 配置变更实时渲染。

### 验收标准

- 用户可选择数据集并绑定字段。
- 图表能根据绑定字段切换展示。
- 清除数据绑定后回到空状态。
- 配置项修改后画布展示同步更新。

## 七、阶段 3：接口数据接入与后端保存联动

时间：第 6-8 周

### 目标

将前端从 Mock 数据推进到真实接口联调阶段，完成真实数据展示、保存配置、读取配置、发布配置的前后端联动。本阶段在第 6 周启动，预留更充足的联调和修正时间，降低后期验收风险。

### 功能模块拆分

#### 1. API Client 接入

- 封装统一请求 client。
- 统一处理：
  - base URL。
  - JSON 请求头。
  - 错误响应。
  - 业务错误码。
  - 网络异常。
- 前端接口调用不直接散落在组件内部。

#### 2. 数据接口联调

- 接入数据集列表接口。
- 接入数据集 schema 接口。
- 接入数据查询接口。
- 使用真实数据验证图表展示。
- 对查询超时、无权限、数据为空、字段缺失做前端降级。

#### 3. 看板配置保存

- 与后端联动保存看板配置。
- 保存内容包括：
  - 看板基础信息。
  - 组件列表。
  - 组件布局。
  - 组件 props。
  - 数据绑定信息。
  - 数据集引用。
  - 主题配置。
- 保存成功后更新 revision。
- 保存失败后保持本地编辑内容。

#### 4. 配置读取与回显

- 打开编辑器时加载后端看板配置。
- 按后端返回内容回显组件布局。
- 回显组件配置项。
- 回显数据绑定字段。
- 回显后触发真实数据查询或保持空状态。

#### 5. 保存冲突处理

- 后端返回版本冲突时弹出冲突提示。
- 用户可选择：
  - 继续编辑本地版本。
  - 重新加载服务端版本。
  - 复制为新看板。
- 冲突处理过程中不丢失当前编辑内容。

#### 6. 发布联动

- 调用发布接口。
- 发布成功后跳转或打开发布页。
- 发布失败时保留旧发布版本。
- 发布状态在工具栏中可感知。

### 主要产出

- 真实接口数据展示。
- 看板配置保存与读取。
- 保存冲突处理。
- 发布接口联动。

### 验收标准

- 真实数据能驱动图表展示。
- 刷新页面后看板配置能正确回显。
- 保存失败或冲突时不会丢失本地编辑内容。
- 发布成功后只读页能展示最新配置。

## 八、阶段 4：预览发布、体验打磨与验收收口

时间：第 9-12 周

### 目标

完成季度末前端验收版本，集中处理交互细节、视觉一致性、异常状态、性能和测试覆盖。由于基础底座已在阶段 1 合并加速完成，本阶段从第 9 周开始进入持续收口，避免体验、异常和测试问题集中堆到最后一周。

### 功能模块拆分

#### 1. 预览体验

- 预览前检查是否有未保存内容。
- 未保存时先保存或提示用户。
- 预览页展示与编辑器保持一致。
- 预览页异常时展示可恢复提示。

#### 2. 发布查看页

- 只读页面去除编辑器操作。
- 发布页布局适配宽屏和常规桌面屏。
- 图表加载失败不影响整页展示。
- 页面不存在时展示明确提示。

#### 3. 交互细节打磨

- 统一 hover、selected、disabled、loading 样式。
- 优化拖拽过程中组件透明度和占位反馈。
- 优化右侧配置项间距和层级。
- 禁用按钮增加说明。
- 长标题、省略号、tooltip 处理。
- 小尺寸图表卡片文字不溢出。

#### 4. 异常状态完善

- 网络失败。
- 接口超时。
- 数据为空。
- 字段缺失。
- Schema 漂移。

## 十三、风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 基础能力合并为第 1 阶段并在两周内完成 | 容易出现底座细节不够精细 | 第 1 阶段只锁定路由、布局、画布、registry 和核心空态，视觉 polish、复杂图表和边缘交互后移 |
| 图表类型多，真实渲染差异大 | 影响阶段交付范围 | 先完成组件分类、空态 demo 和基础数据态，复杂图表逐步增强 |
| 拖拽碰撞和布局边界复杂 | 容易出现重叠、跳动 | 优先稳定网格算法和验收用例，再做视觉动效 |
| 后端接口进度不确定 | 影响真实数据调测 | 前端先用 MSW 契约开发，后续按 OpenAPI 切真实接口 |
| 配置项过多 | 右侧面板复杂度上升 | 先做通用配置，再做高频图表差异化配置 |
| 真实数据异常场景多 | 容易造成图表空白或报错 | 所有数据链路补空态、错误态和局部错误边界 |
| 后期体验问题集中暴露 | 验收风险升高 | 第 9 周开始进入持续收口，每周处理体验、异常和测试问题，不把 polish 全部堆到最后 |
